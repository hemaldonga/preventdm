# Documentation

This folder contains architectural documentation and design artifacts for the PreventDM project.

## Diagrams

The system is documented through three foundational diagrams.

The system architecture diagram shows the five-tier component structure of the application, including the user roles, the Flutter frontend, the FastAPI backend, the Rust and ML services, and the data layer. See `system-architecture.svg`.

The data flow diagram traces the runtime sequence of a patient risk assessment as it moves through the four backend services and into the database. See `data-flow-diagram.svg`.

The entity-relationship diagram describes the data model, including the nine entities and their relationships, with full attribute definitions for each table. See `entity-relationship-diagram.md`.

Additional documentation will be added as the project develops, including the technical architecture document, the product brief, the regulatory and ethics analysis, and the deployment guide.
